# Design PRD: 맛집 저장 및 추천 앱 디자인 컨셉

## 1. 디자인 비전

### 1.1 핵심 방향성

> **"따뜻한 미니멀리즘 — 내 손 안의 맛집 노트"**

개인 맛집을 기록하는 **아날로그적 감성**과, 빠르게 추천받는 **디지털 효율성**을 동시에 전달하는 디자인.
화려함 대신 **편안함**, 복잡함 대신 **직관성**을 추구한다.

### 1.2 디자인 원칙

| 원칙 | 설명 | 적용 예시 |
|------|------|----------|
| **Calm & Warm** | 눈이 편안하고 따뜻한 톤으로 식욕과 안정감을 동시 자극 | Warm Neutral 배경, 부드러운 그림자 |
| **Content First** | 식당 사진과 정보가 주인공, UI 요소는 뒤로 | 큰 이미지 카드, 최소한의 장식 요소 |
| **One-Tap Action** | 핵심 동작은 한 번의 탭으로 완료 | 추천 버튼, 저장 버튼의 즉각적 접근 |
| **Delightful Micro-Motion** | 목적이 있는 마이크로 인터랙션으로 피드백 제공 | 저장 시 바운스, 별점 채워지는 애니메이션 |
| **Accessible by Default** | 접근성은 선택이 아닌 기본 | 명확한 대비, 충분한 터치 영역, 다크 모드 |

## 2. 트렌드 분석 & 적용

### 2.1 반영하는 2025-2026 트렌드

| 트렌드 | 업계 동향 | 우리 앱 적용 |
|--------|----------|-------------|
| **Warm Minimalism** | 차가운 회색 미니멀에서 따뜻한 뉴트럴 톤으로 이동 | Sand/Cream 기반 배경, 부드러운 곡선 |
| **Glassmorphism (Liquid Glass)** | Apple iOS 26에서 Liquid Glass 도입, 반투명 UI 주류화 | 추천 카드, 바텀 시트에 frosted glass 효과 적용 |
| **Purposeful Micro-Interactions** | 단순 장식이 아닌 기능적 피드백 목적의 모션 | 저장/삭제/평점 입력 시 의미 있는 애니메이션 |
| **AI-Driven Personalization** | 사용자 맥락 기반 적응형 UI | 시간대별 추천 UI 변화 (점심/저녁), 자주 쓰는 카테고리 상단 배치 |
| **Dark Mode** | 선택이 아닌 필수, 커스터마이즈 가능한 테마 | Light/Dark/System 3가지 모드 지원 |
| **Bento Box Layout** | 다양한 크기의 카드를 조합한 대시보드 스타일 | 홈 화면에 카테고리 칩 + 추천 카드 + 최근 저장 믹스 레이아웃 |
| **Soft Depth (Neo-Skeuomorphism)** | 부드러운 그림자와 질감으로 깊이감 | 카드 컴포넌트에 layered shadow, 살짝 눌리는 버튼 |
| **Passwordless Auth** | 생체인증, 매직링크 등 비밀번호 없는 인증 | Face ID/Touch ID 자동 로그인, 소셜 로그인 우선 |

### 2.2 의도적으로 배제하는 트렌드

| 트렌드 | 배제 이유 |
|--------|----------|
| 3D/AR 요소 | MVP 복잡도 증가, 맛집 앱에서 실질적 가치 낮음 |
| 음성 인터페이스 | 개인 기록 앱 특성상 텍스트 입력이 더 적합 |
| 과도한 애니메이션 | 기록과 탐색 속도를 저해, 성능 이슈 유발 |
| 네온/비비드 컬러 | 음식/맛집 앱의 따뜻한 감성과 불일치 |

### 3.2 컬러 적용 원칙

- **60-30-10 Rule**: 배경(Cream/Dark) 60%, 카드/서피스(Sand) 30%, 액센트(Tangerine) 10%
- **음식 사진 우선**: 컬러풀한 음식 사진이 돋보이도록 UI 색상은 절제
- **대비율**: WCAG AA 기준 충족 (일반 텍스트 4.5:1, 큰 텍스트 3:1)

## 4. 타이포그래피

### 4.1 폰트 선정

| 용도 | 폰트 | 선정 이유 |
|------|------|----------|
| **한글 본문** | Pretendard | 가독성 우수, 다양한 웨이트, 무료 |
| **한글 강조** | Pretendard Bold/SemiBold | 동일 서체 패밀리로 통일감 |
| **영문/숫자** | Pretendard (내장 Latin) | 한/영 혼용 시 조화 |
| **숫자 강조** | SF Pro (iOS) / Roboto (Android) | 평점, 거리 등 숫자 정보 가독성 |

### 4.2 타입 스케일

```
Display    28px / Bold    / -0.02em  ← 추천 카드 식당 이름
Title 1    24px / Bold    / -0.01em  ← 화면 제목
Title 2    20px / SemiBold / 0em     ← 섹션 제목
Body 1     16px / Regular  / 0em     ← 본문 텍스트
Body 2     14px / Regular  / 0.01em  ← 보조 텍스트, 메모
Caption    12px / Medium   / 0.02em  ← 태그, 타임스탬프, 라벨
Overline   11px / SemiBold / 0.05em  ← 카테고리 칩 텍스트
```

### 4.3 Line Height

| 타입 | Line Height | 용도 |
|------|------------|------|
| Compact | 1.2 | 제목, 카드 내 짧은 텍스트 |
| Normal | 1.5 | 본문, 리뷰 내용 |
| Relaxed | 1.7 | 긴 설명, 메모 입력 영역 |

```

spacing-3:   12pt    ← 그룹 내 요소 간격
spacing-4:   16pt    ← 섹션 내 요소 간격, 카드 패딩
spacing-5:   20pt    ← 카드 간 간격
spacing-6:   24pt    ← 섹션 간 간격
spacing-8:   32pt    ← 큰 섹션 간 구분
spacing-10:  40pt    ← 화면 상단 여백
```

## 6. 아이콘 & 일러스트레이션

### 6.1 아이콘 스타일

| 속성 | 값 |
|------|-----|
| 스타일 | Outlined (1.5pt stroke), Rounded corners |
| 크기 | 20pt (리스트), 24pt (탭바/액션), 28pt (강조) |
| 컬러 | 단색 (Stone 700 기본, Tangerine 500 활성) |
| 라이브러리 | Phosphor Icons (React Native 호환, 일관된 스타일) |

### 6.2 핵심 아이콘 목록

```
탭바:
  홈        → House
  추천       → Compass (또는 Sparkle)
  프로필     → User

액션:
  추가       → Plus
  검색       → MagnifyingGlass
  필터       → Funnel
  정렬       → SortAscending
  공유       → Share
  편집       → PencilSimple
  삭제       → Trash
  카메라     → Camera
  위치       → MapPin
  전화       → Phone
  별점       → Star (filled/outlined)
  카테고리   → Tag
  뒤로가기   → CaretLeft
  닫기       → X
```

### 6.3 Empty State 일러스트

빈 상태에는 **단색 라인 일러스트** 사용:

| 상태 | 일러스트 | 메시지 |
|------|---------|--------|
| 저장된 식당 없음 | 빈 접시 + 포크/나이프 | "아직 저장한 맛집이 없어요\n첫 번째 맛집을 추가해보세요" |
| 검색 결과 없음 | 돋보기 + 물음표 | "검색 결과가 없어요\n다른 키워드로 찾아보세요" |
| 추천 불가 | 지도 핀 + 반경 원 | "주변에 저장한 맛집이 없어요\n맛집을 더 저장해보세요" |
| 리뷰 없음 | 연필 + 노트 | "아직 리뷰가 없어요\n방문 후 기록을 남겨보세요" |
| 네트워크 오류 | 끊어진 와이파이 | "인터넷 연결을 확인해주세요" |

## 9. 모션 & 인터랙션

### 9.1 모션 원칙

| 원칙 | 설명 |
|------|------|
| **빠르고 자연스럽게** | 대부분의 전환은 200-300ms |
| **목적이 있는 모션만** | 상태 변화를 알리거나 공간감을 주는 모션만 사용 |
| **과하지 않게** | 같은 애니메이션이 반복되면 피로, 최소화 |

### 9.2 핵심 인터랙션

| 인터랙션 | 애니메이션 | Duration | Easing |
|----------|-----------|----------|--------|
| 화면 전환 | Shared element transition (카드→상세) | 300ms | ease-out |
| 카드 탭 | Scale 0.97 → 1.0 (press feedback) | 150ms | ease-out |
| 식당 저장 | Checkmark lottie + 카드 slide-in | 400ms | spring |
| 별점 입력 | Star fill + scale bounce | 150ms | spring |
| 추천 카드 스와이프 | 카드 가로 슬라이드 + 페이드 | 250ms | ease-in-out |
| "여기로 결정" | Confetti burst + haptic feedback | 600ms | spring |
| 삭제 | 카드 slide-left + fade-out | 200ms | ease-in |
| Pull to refresh | Custom lottie (포크 회전) | - | - |
| Bottom sheet 등장 | Slide-up + backdrop fade | 300ms | ease-out |
| 추천 셔플 | 카드 3D flip + reorder | 400ms | spring |

### 9.3 Haptic Feedback

| 이벤트 | Haptic 유형 |
|--------|-------------|
| 버튼 탭 | Light Impact |
| 식당 저장 | Success Notification |
| 삭제 확인 | Warning Notification |
| 별점 선택 | Light Impact (각 별마다) |
| 추천 결정 | Success Notification |
| 에러 | Error Notification |

| 항목 | 기준 | 적용 |
|------|------|------|

| 컬러 의존 제거 | 아이콘/텍스트 병행 | 색상만으로 정보 전달하지 않음 |

### 11.2 다크 모드 변환 규칙

| Light Mode | Dark Mode | 설명 |
|------------|-----------|------|
| Cream 50 (배경) | Dark 950 | 앱 배경 |
| Sand 100 (카드) | Dark 900 | 카드/서피스 |
| Sand 200 (구분선) | Dark 800 | 구분선 |

| Tangerine 500 (액센트) | Tangerine 400 | 밝기 보정 |

- Glassmorphism 블러 강도 다크 모드에서 조정 (blur 약간 강하게)
